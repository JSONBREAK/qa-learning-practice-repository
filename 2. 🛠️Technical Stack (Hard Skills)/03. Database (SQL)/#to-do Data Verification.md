# Data Verification (SQL) สำหรับ QA

## ทำไมต้องเช็คข้อมูลหลังบ้าน
- UI ผ่าน แต่ข้อมูลอาจไม่ถูกเก็บ
- ต้องยืนยันว่า Database สอดคล้องกับสิ่งที่เห็นบนหน้าจอ

## ต้องเช็คอะไรบ้าง
- **Insert**: ข้อมูลถูกสร้างจริงไหม
- **Update**: แก้แล้วอัปเดตถูกแถวไหม
- **Delete**: ลบแล้วหายจริงไหม
- **Relationship**: ตารางที่เกี่ยวข้องอัปเดตครบไหม

## คิวรี่พื้นฐานที่ใช้บ่อย
- SELECT + WHERE
- COUNT
- JOIN
- ORDER BY / GROUP BY

## Checklist แบบสั้น
- [ ] มี record ใหม่ตามที่คาด
- [ ] ค่า field ถูกต้อง
- [ ] ไม่มี record ซ้ำหรือค้าง
- [ ] ตารางที่เกี่ยวข้องอัปเดตครบ
